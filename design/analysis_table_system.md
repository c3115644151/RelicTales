# 分析台与陶片知识树系统设计

> 版本：v0.4
> 状态：计算模型已确定

---

## 一、核心方块：分析台 (Analysis Table)

### 合成配方
3 石砖 + 2 纸 + 1 羽毛 + 1 墨囊 → 1 分析台

### GUI 布局示意

```
[陶片] [陶片] [陶片] [陶片] [陶片]     ← 陶片槽（可容纳 1-5 片）
                [纸]                     ← 基础消耗品
                [/]                      ← 文字指示线
                [消耗品组合]             ← 根据陶片数量自动判断所需层级
           ↓↓↓↓↓ 分析按钮 ↓↓↓↓↓
            [产出槽]
```

### 核心原则

- 陶片本身**永不消耗**，只有辅助材料消耗
- 分析成功与否由消耗品是否满足条件决定——不设"分析失败"，消耗品放入即启动计算
- 消耗品决定分析深度，陶片决定产出方向

## 二、消耗品三级阶梯

| 分析类型 | 所需消耗品 | 成本换算 |
|---|---|---|
| 1 片 | 纸 ×1 | 3 甘蔗 |
| 2-3 片同遗迹 / 任意 2 片 | 纸 ×3 + 墨囊 ×1 | 9 甘蔗 + 1 鱿鱼 |
| 4-5 片同遗迹 / 跨遗迹组合 | 书与墨囊 ×1 | 1 皮革 + 3 纸 + 1 墨囊 |
| 跨遗迹组合（高级） | 书与墨囊 ×1 + 绿宝石 ×1 | 上书 + 1 绿宝石 |

不同分析类型消耗品不互通——放 3 片但只放纸，分析按钮灰色不可用。

| 维度 | 含义 | 覆盖的配方方向 |
|---|---|---|

| **Craftsmanship (工)** | 建筑、冶炼、装饰、工具 | 特殊砖块、染料、工具附魔 |

### 3.3 属性聚合规则

多陶片分析时各维度的值**逐维求和**。同一个陶片品种最多只能放入一片（不区分来源地，只区分陶片品种）。

例：沙漠 3 片（珍宝纹样陶片 ×1 + 弓箭纹样陶片 ×1 + 采矿纹样陶片 ×1）：
```
{商3, 农1, 学1} + {野4, 工1} + {工2, 学2, 农1} = {农:2, 工:3, 学:3, 商:3, 野:4}
```

## 四、配方匹配算法

### 4.1 配方定义

每个配方只包含两个字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| **需求向量 R** | `{dim: value, ...}` | 在 5 维度上的需求值。**无上限**（简单配方 6-8，顶级配方可达 25+） |
| **产出** | `{type, id}` | 配方（玩家学习后制作）或 分析消耗品（分析台直接产出） |

### 4.2 匹配规则（两道门禁）

```
规则一：维度门禁（Binary Gate）
  对 R 中每个非零维度 d，I_d 必须 > 0
  → 否则该配方直接淘汰，不进入候选池

规则二：匹配度门禁（M ≥ 0.4）
  M = sum(min(I_d, R_d) for each d where R_d > 0) / sum(R)
  → M < 0.4 淘汰
  → M ≥ 0.4 进入候选池
```

### 4.3 加权随机抽取

```
候选池中每个配方 i 的选中概率 P_i = M_i² / sum(M_j² for j in candidate pool)
```

单一决策变量 M 决定一切。无额外品级表、无多阶段投骰。

### 4.4 关键行为推演

**场景 A：单陶片分析**

输入沙漠珍宝纹样陶片 = {农:1, 学:1, 商:3}，共 3 个维度有值。

- 配方"生长灵药"需求 {农:8} → 门禁通过（农有值），M = 1/8 = 0.125 → < 0.4，淘汰
- 配方"沙漠黄染料"需求 {商:2} → 门禁通过，M = 2/2 = 1.0 → 进池，权重 1.0
- 配方"古代贸易路线图"需求 {商:4, 学:2} → 门禁通过，M = (3+1)/(4+2) = 0.67 → 进池，权重 0.45
- 配方"远古共鸣石"需求 {农:3, 工:3, 学:3, 商:3, 野:3} → 门禁不通过（工=0，野=0） → 淘汰

结果：候选池极小（2-3 个），产出高度随机，但方向锁定在沙漠相关。

**场景 B：同遗迹 5 片全放**

沙漠 5 片聚合 = {农:2, 工:6, 学:5, 商:5, 野:7}，5 个维度全覆盖。

- "生长灵药"{农:8} → M = 2/8 = 0.25 → 淘汰（沙漠农值低）
- "沙漠黄染料"{商:5} → M = 5/5 = 1.0 → 进池，权重 1.0
- "古代贸易路线图"{商:7, 学:4} → M = (5+4)/(7+4) = 0.82 → 进池，权重 0.67
- "远古共鸣石"{农:5, 工:5, 学:5, 商:5, 野:5} → 门禁通过（5维都有值），M = (2+5+5+5+5)/(5+5+5+5+5) = 22/25 = 0.88 → 进池，权重 0.77

结果：候选池 10+ 个，高匹配配方权重占优。"生长灵药"因沙漠农业弱势而被淘汰——即使 5 片全放也不够。"远古共鸣石"M=0.88 进了池，但它更高的 R 值（总和 25）意味着跨遗迹组合才能达到完美 M=1.0，单遗迹始终存在维度和值上限的瓶颈。

**场景 C：跨遗迹组合——天然的高维覆盖**

沙漠 3 片 + 要塞 2 片聚合 = {农:2, 工:6, 学:8, 商:6, 野:4}，5 维全覆盖，且各维度值更均衡。

- "远古共鸣石"{农:5, 工:5, 学:5, 商:5, 野:5} → M = (2+5+5+5+4)/25 = 21/25 = 0.84 → 进池，权重 0.71
- "传送共鸣附魔"{学:14, 商:5, 野:4} → M = (8+5+4)/23 = 17/23 ≈ 0.74 → 进池，权重 0.55
- 对比：纯要塞 5 片（{学:14, 商:4}，野=0）→ **门禁不通过**，传送共鸣附魔不可解锁

跨遗迹组合的候选池更大、权重更高、覆盖更广。这不是额外规则造成的，是数学本身——单遗迹总有维度短板，跨遗迹才能填补缺失。

## 五、产出类型

### 5.1 永久配方

| 配方 | 需求向量 R | 预期负载 |
|---|---|---|
| 古代染料（6 色） | {工:6} | 1-2 片即可（工≥3 即达标） |
| 古法鱼饵 | {野:6} | 1 片丛林尖牙纹样陶片（野:5）即 M=0.83 |
| 生长灵药 | {农:8} | 2-3 片丛林/Trail Ruins |
| 勘探熏香 | {野:8, 学:5} | 3-4 片交叉维度 |
| 遗迹罗盘 | {学:6, 商:5} | 2-3 片要塞 |
| 知识粉尘 | {学:8} | 2-3 片要塞（单要塞片不够） |
| 刷子专属附魔 | {学:8, 工:7} | 3-4 片要塞/Trail Ruins |

### 5.2 分析台直接产出的消耗品

| 物品 | 获取条件 | 功能 |
|---|---|---|
| **交易特许状** | 商 ≥4 的输入向量 | 赠予村民 → 永久解锁隐藏交易项。该特定村民每天可交易 1 次。需村民已达最高等级（Master） |
| **嗅探兽训练粮** | 农 ≥3 的输入向量 | 喂食嗅探兽 → 30 分钟内每次挖掘额外概率产出稀有古代种子（原版产出保留） |

### 5.3 分类原则

- **配方** = 学会后**玩家自行制作**，可量产，功能性
- **分析消耗品** = 分析台直接产出，**不可制作**，高叙事价值，1 次性使用
- 鉴定标准："这个物品如果批量生产，体验会变好还是变差？"——如果批量生产会让体验变差，就做成分析消耗品

## 八、村民交易联动

- 交易特许状（分析消耗品）：赠予村民 → 该村民**永久**解锁一组隐藏交易项
- 限制条件：该村民必须已达最高等级（Master，第 5 级）
- 频率限制：解锁的交易项每天只能交易 1 次（MC 原版村民的 daily supply 机制）
- 不同来源的特许状解锁不同交易项：

| 特许状来源 | 适用村民 | 示例隐藏交易 |
|---|---|---|
| 沙漠 | 制图师 | 出售"考古地图"（指向未探索的 Trail Ruins / 远古城市） |
| 丛林 | 牧羊人 / 农民 | 出售稀有古代种子 |
| 要塞 | 图书管理员 | 出售刷子专属附魔书 |
| Trail Ruins | 石匠 | 出售古代装饰方块 |
| 下界 | 牧师 | 出售稀有下界物品（如特定附魔的火焰保护书） |
| 海洋 | 渔夫 | 出售特殊钓鱼竿附魔 |

## 十、已解决 / 已确定的决策

| 议题 | 决策 |
|---|---|
| 消耗品阶梯 | 纸 → 纸×3+墨囊×1 → 书与墨囊×1（详见第二章） |
| 偏置值机制 | **去掉**，每次分析独立，无累积效应 |
| 跨遗迹组合 | 不设特殊规则，**属性模型天然覆盖**——高维配方只有跨遗迹才能满足门禁条件 |
| 配方品级 | **去掉**配方固定品级和品级概率表——M 是唯一决策变量 |
| 结构标签过滤 | **去掉**——维度门禁天然替代了结构过滤功能 |
| 产出分类 | 配方（可学可量产）vs 分析消耗品（不可制作，高叙事）已明确 |
| 交易特许状 | 永久解锁特定村民隐藏交易，需村民已达 Master，每天可交易 1 次 |
