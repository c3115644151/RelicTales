## 一、项目概述

### 1.1 基础信息

| 属性           | 值                                |
| ------------ | -------------------------------- |
| 模组名称         | RelicTales（考古物语）                 |
| Mod ID       | `relictales`                     |
| Minecraft 版本 | 26.1.1（年进制版本）                    |
| Mod Loader   | NeoForge 26.1.1.14-beta          |
| Java 版本      | 25（Foojay resolver 自动管理）         |
| 核心主题         | 考古发掘 + 反工业化 + 沉浸式诅咒体验            |
| 目标整合包        | 以内容深度而非科技树取胜，目标是成为整合包中的「考古主题填充包」 |

### 1.2 核心体验描述

> 玩家在世界的深层古老地层中，通过錾子与刷子的传统工具组合，发掘埋藏的遗物与化石。每件遗物都承载着古老的诅咒，持有越久诅咒越深，最终需要通过镇魂祭坛净化。诅咒通过视觉/音效的递进式反馈给玩家，而非粗暴的数值惩罚。

### 1.3 设计哲学

| 原则          | 说明                             |
| ----------- | ------------------------------ |
| **反工业化**    | 无自动化机器、无流水线；所有交互必须玩家手动操作       |
| **沉浸式反馈**   | 通过视听反馈（粒子、音效、视觉暗角）传达状态，不依赖数值UI |
| **风险-回报对等** | 高稀有度遗物诅咒积累更快，但功能更强             |
| **传统工具链**   | 錾子（单一类型）+ 刷子（四级递进），不做工具升级树     |
| **内容即深度**   | 通过大量遗物种类、稀有度变种、诅咒阶段组合实现重玩性     |

## 三、工具系统

### 3.1 錾子（Chisel）

**设计说明：** 单一类型工具，用于处理硬质考古方块（可疑的沉积岩/暗曜岩）。

| 属性   | 值                            | 状态   |
| ---- | ---------------------------- | ---- |
| 物品ID | `relictales:chisel`          | ✅ 确认 |
| 物品类型 | Tools（可附魔）                   | ✅ 确认 |
| 合成配方 | 2铜锭 + 1木棍（V形，工作台）            | ✅ 确认 |
| 耐久值  | 120（铜工具等级）                   | ✅ 确认 |
| 可附魔  | 耐久I/II/III（不可附魔效率）           | ✅ 确认 |
| 敲击次数 | 矿物决定（浅层2次，深层4次）              | ✅ 确认 |
| 交互模式 | 每次敲击 → 硬直0.4秒（原版铁砧延迟） → 状态更新 | ✅ 确认 |

**敲击状态机（通用）：**

```
STATE_0（完整）→ STATE_1（轻微裂纹）→ STATE_2（中度裂纹）→ STATE_3（密集裂纹）→ STATE_BRUSHPABLE（可刷）
                                                                 ↑
                                                         敲击次数由矿物决定
```

**视觉效果：** 每次敲击产生粒子（岩石碎片）+ 音效变化（裂纹加深）。

**⚠️ 已确认：独立JSON方案** — 每个敲击阶段有独立的 .json 模型文件 + .png 贴图，NeoForge 原生支持，无需自定义渲染器。

### 3.2 刷子（Brush）

**设计说明：** 五级刷子（铜/铁/金/钻石/下界合金），按矿物类型匹配效率。铜/铁/金/钻石刷子通过工作台合成，配方参照原版（1锭 + 1棍）；下界合金刷子通过锻造台升级（钻石刷子 → 下界合金刷子）。

| 物品ID                         | 合成配方                 | 耐久值  | 效率倍率 | 适用矿物    | 状态   |
| ---------------------------- | -------------------- | ---- | ---- | ------- | ---- |
| `relictales:copper_brush`    | 1铜锭 + 1木棍（工作台）       | 64   | 1.0x | 沉积石灰岩   | ✅ 确认 |
| `relictales:iron_brush`      | 1铁锭 + 1木棍（工作台）       | 250  | 1.5x | 沉积页岩/板岩 | ✅ 确认 |
| `relictales:gold_brush`      | 1金锭 + 1木棍（工作台）       | 32   | 2.0x | 暗曜岩     | ✅ 确认 |
| `relictales:diamond_brush`   | 1钻石 + 1木棍（工作台）       | 512  | 2.5x | 所有矿物    | ✅ 确认 |
| `relictales:netherite_brush` | 钻石刷子 + 1下界合金锭（锻造台升级） | 2031 | 3.0x | 所有矿物    | ✅ 确认 |

**原版刷子冲突问题：**

> 原版MC有铜刷子（铜锭+木棍），配方与我们的铜刷子完全重叠。
> **处理方案：** 原版铜刷子保留在游戏中，不做删除。玩家可以同时拥有两把铜刷子（功能完全相同），仅作为共存状态，不做特殊处理。

**效率倍率含义：** 影响每次刷取贡献的「进度点」。

```
矿物刷取总点数 = 矿物基础刷取次数（如3次）
实际需要刷取次数 = ceil(矿物刷取点数 / 刷子效率倍率)

例：暗曜岩基础需刷3次
  - 铜刷子：ceil(3/1.0) = 3次
  - 铁刷子：ceil(3/1.5) = 2次
  - 金刷子：ceil(3/2.0) = 2次
  - 钻石刷子：ceil(3/2.5) = 2次
  - 下界合金刷子：ceil(3/3.0) = 1次
```

- 可出现在矿洞暴露面、洞穴墙壁、峡谷岩壁等**任何符合条件的石头表面**
- 生成时替代石头/深板岩，不挑环境

```
生成参数（待调整，需测试平衡性）：

│ 生成数量: 每个chunk生成 1~3 个斑块                           │
│ 可疑比例: 斑块内约 15~20% 为可疑变种                         │
│ 生成高度: 矿物各自定义的Y轴区间                              │
│ 生成检查: 替代石头/深板岩类型，不生成在空气/水下/岩浆旁        │
```

### 4.3 可疑方块识别机制

**方案：粒子特效标识**

可疑方块带有 **持续性的微小粒子效果**（类似灵魂碎片飘动），在不破坏整体视觉一致性的前提下提供「这里有东西」的提示。

### 5.2 第三形态说明

进入第三形态的方块：

- 视觉上有明显变化（更强光芒/更多粒子）
- 显示为「已露出核心」状态
- 物品已经在方块内实体化，等待取出

### 5.3 蛮力破坏惩罚

| 条件                 | 惩罚类型 | 概率  | 说明           |
| ------------------ | ---- | --- | ------------ |
| 无slimeball破坏第三形态方块 | 蠹虫生成 | 30% | 蠹虫生成后会主动攻击玩家 |
| 无slimeball破坏第三形态方块 | 遗物损坏 | 50% | 遗物消失，不掉落任何东西 |

## 六、诅咒系统

### 6.1 诅咒值属性

| 属性    | 值                         | 说明               |
| ----- | ------------------------- | ---------------- |
| 诅咒值范围 | 0 \~ 100                  | 对玩家不可视数值         |
| 衰减媒介  | 屏幕边缘暗角强度                  | 玩家通过暗角深浅感知诅咒积累程度 |
| 衰减速度  | 持有无诅咒遗物 -0.3/秒；无遗物 -0.5/秒 | 线性衰减             |
| 积累速度  | 持有遗物时按稀有度积累（见下表）          | 非线性，按秒积累         |

**稀有度-诅咒积累速度对应表（已除以10，降低积累速度）：**

| 稀有度 | 积累速度    | 说明            |
| --- | ------- | ------------- |
| 普通  | +0.05/秒 | 极缓慢，10秒+才感知变化 |
| 稀有  | +0.1/秒  | 缓慢            |
| 传说  | +0.15/秒 | 中等            |
| 史诗  | +0.2/秒  | 较快            |

### 6.2 诅咒阶梯惩罚

| 诅咒值区间  | 视觉特效           | 音效暗示                 | 机制影响                           |
| ------ | -------------- | -------------------- | ------------------------------ |
| 0-30   | 屏幕边缘轻微暗角       | 微弱低频嗡鸣（安静环境可闻）       | 无                              |
| 31-50  | 暗角加深，偶尔有粒子飘过视野 | 随机方向的低语耳语声           | 轻微：行走时偶发视角轻微抖动                 |
| 51-70  | 暗角脉动，有呼吸感      | 清晰耳语 + 隐约脚步声跟随       | 中等：5%概率/分钟出现假「敌意检测」提示（屏幕边缘闪红点） |
| 71-85  | 暗角持续加深，视野边缘变色  | 清晰脚步声持续跟随，远处有呼唤名字的幻听 | 较强：背包内部分物品图标出现「幽灵化」（视觉闪烁）      |
| 86-100 | 几乎全屏暗角，仅中心可见   | 持续尖锐耳鸣 + 持续呼唤声       | 严重：假受伤体验（屏幕闪红+心形扣减动画，但实际不减生命值） |

### 6.3 追魂（幻觉）机制

在 **71-85 诅咒值区间** 触发：

- 在玩家 **左肩方向**（或身后方向）生成一个跟运动方向一致的暗影幻觉
- 永远保持在视野边缘，不会在正前方出现
- 玩家**停下脚步**，幻觉逐渐消散
- 玩家**开始奔跑**，幻觉「追得更紧」（暗角收窄）
- 纯视觉幻觉，无碰撞、无实体、无伤害

### 6.4 诅咒衰减机制细节

| 场景      | 诅咒值变化   | 衰减/积累速率 |
| ------- | ------- | ------- |
| 持有普通遗物  | +0.05/秒 | 积累      |
| 持有稀有遗物  | +0.1/秒  | 积累      |
| 持有传说遗物  | +0.15/秒 | 积累      |
| 持有史诗遗物  | +0.2/秒  | 积累      |
| 持有净化后遗物 | -0.01/秒 | 极缓慢衰减   |
| 身上无遗物   | -0.05/秒 | 自然衰减    |

## 七、净化系统

### 7.1 镇魂祭坛（Soul Altar）

| 属性   | 值                             |
| ---- | ----------------------------- |
| 方块ID | `relictales:soul_altar`       |
| 外观   | 类似营火 + 紫色/蓝色火焰 + 祭坛底座         |
| 合成配方 | 3石头 + 1紫水晶 + 1灵魂土（类似灵魂营火）     |
| 成本   | 1紫水晶（每次净化）                    |
| 净化方式 | 异步完成（可取消）                     |
| 完成时间 | 30秒（600 ticks）                |
| 交互流程 | 放置遗物 → 放置紫水晶 → 开始净化 → 完成后玩家可取 |

**交互状态机：**

```
空载状态 → 放置遗物 → 放置紫水晶 → 开始净化（火焰变色+粒子聚集，紫水晶开始逐渐消耗）
→ 净化中（可被取消）→ 完成后 → 遗物变为净化状态 → 玩家可取
```

**取消机制（关键设计）：**

| 场景          | 结果                         |
| ----------- | -------------------------- |
| 净化中玩家右键取出遗物 | 净化中断，遗物保持诅咒状态，紫水晶消失，净化进度丢失 |
| 净化完成前不取出    | 30秒后自动完成，遗物净化成功            |

> **设计理由：** 可取消的设计让玩家有「后悔药」，避免因误操作导致的紫水晶浪费。但取消的代价（紫水晶消失）确保了决策的风险性。

## 十六、关键设计决策备忘

### 决策1：废除敲碎机制

**日期：** 2026/04/08

**问题：** 錾子敲击超过上限是否导致方块碎裂（产出丢失）？

**结论：** 废除。敲击超过上限仅停留在最终状态，不碎裂，不丢失产出。

**理由：** 如果敲碎风险存在，玩家会通过刷子试探敲击是否到位——这使敲击变成「试错」而非「操作」，破坏了操作的心流体验。

***

### 决策2：刷子不设敲碎风险

**日期：** 2026/04/08

**问题：** 刷取时是否有过快导致遗物损坏的风险？

**结论：** 无刷碎风险。

**理由：** 考古流程已经够长（敲4-5次 + 刷3-5次 + 判定），再增加风险会使整体体验过于硬核，违背「反工业化但不反乐趣」的设计原则。

***

### 决策3：诅咒值不可视但有感知媒介

**日期：** 2026/04/08

**问题：** 诅咒值对玩家完全不可见，如何避免玩家把诅咒惩罚当成Bug？

**结论：** 视觉暗角作为诅咒值的「感知代理」，使诅咒积累可感知但无精确数值。

**理由：** 数值UI会破坏沉浸感，但完全无反馈会让诅咒系统变成「意外的惩罚」。暗角提供渐变的视觉反馈，让玩家能关联自身状态与诅咒值。

***

### 决策4：单一錾子 + 五级刷子

**日期：** 2026/04/08

**问题：** 錾子是否需要分级？刷子是否需要分级？

**结论：** 錾子不分级（单一类型），刷子分五级（铜/铁/金/钻石/下界合金）。钻石→下界合金通过锻造台升级，其余通过工作台合成。

**理由：** 錾子的核心交互是「判定敲击次数」而非「升级工具能力」，分级无意义且增加复杂度。刷子分级（且铜/铁/金/钻石沿用原版合成模式）让玩家有自然的升级路径，同时与矿物难度匹配的设计契合。
